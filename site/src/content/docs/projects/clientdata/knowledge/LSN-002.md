# LSN-002: Astryx migration — lockfile must be updated before deploy

## Lesson

After adding Astryx packages and rewriting all components, the Vercel build failed because `pnpm-lock.yaml` was stale. Vercel uses `--frozen-lockfile` by default. The fix was running `pnpm install` locally to regenerate the lockfile before deploying.

Same issue happened again when removing `next-themes` from `package.json` — the lockfile change wasn't committed with the package.json change. `git status --short` would have shown `M pnpm-lock.yaml`.

## Prevention

After any `npm install`/`pnpm add`, immediately commit or at minimum regenerate the lockfile. Always run `npm run build` locally to catch lockfile-related failures before pushing to CI.
