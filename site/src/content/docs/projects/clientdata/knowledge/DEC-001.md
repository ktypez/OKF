# DEC-001: SPA with History API routing

## Decision

Use a single-page application pattern with History API (`pushState`/`popState`) instead of Next.js full page navigation for the main dashboard.

## Rationale

- Avoids full page reloads when switching between Map/List/Admin views
- Preserves map state (zoom, center, pins) across view switches
- Simpler state management for a dashboard that's always mounted

## Consequences

- Must manually manage browser history stack
- View state is lost on hard refresh (mitigated by localStorage caching)
- SEO doesn't apply to dashboard (admin-only area)
