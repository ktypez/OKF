# Code Review

Systematic code review across 5 dimensions: quality, bugs, security, performance, and best practices.

## Coverage

| Dimension | Focus Areas |
|-----------|-------------|
| Code Quality | Smells, anti-patterns, duplication, naming, file org |
| Bug Detection | Logic errors, edge cases, null handling, async error swallowing |
| Security | Input validation, auth gating, secrets exposure |
| Performance | Re-renders, memoization, bundle size, memory leaks |
| Best Practices | Codebase consistency, error handling, test coverage |

## Usage

Invoke via opencode skill system — triggered by "code review" in conversation or used as part of cleanup workflow.
