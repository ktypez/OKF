# DEC-007: Split loading gate for instant app shell rendering

## Decision

Instead of blocking the entire page behind `LoadingScreen` until data loads, render the app shell (sidebar + header) immediately and only skeleton the content area.

## Rationale

- Users see the real app structure instantly (sidebar, header) instead of a full-page skeleton
- Content area skeleton is lightweight (toolbar + table rows only)
- `initialLoadDone` ref + sessionStorage still prevents skeleton on subsequent SPA navigations
- All `dynamic()` loading callbacks changed to `null` — no component bundling triggers a full-page skeleton flash

## Implementation

- `page.tsx` line 583: Removed `if (loading && !initialLoadDone.current) return <LoadingScreen />`
- Content area (list view): `loading && !initialLoadDone.current ? <Skeleton+TableSkeleton /> : <real content />`
- `layout.tsx`: Added inline `<script>` in `<head>` for branded splash (dark bg + centered pulsing icon, auto-removes on load)
- Deleted `app/icon.svg` (conflicted with `favicon.svg` in metadata)
