# RSK-001: View state lost on hard refresh (FIXED)

## Risk

The SPA uses in-memory React state for the current view (map/list/admin). A hard browser refresh resets this to the default view, losing the user's context.

## Resolution

Fixed 2026-07-05 — added URL param restoration on mount:

- On load, reads `window.location.search` params (`?detail`, `?map`, `?add`, `?edit`, `?suggestions`) and restores the corresponding view
- `search`, `filter`, and `viewMode` persisted to `localStorage` and restored on refresh
- Restoration runs after `useClients()` finishes loading (clients data available)
- State writes wrapped in `startTransition` for React 19 compatibility

## Mitigation (historical)

- `localStorage` caching of selected client, current view, and map center
- URL hash parameter for view state (planned)
