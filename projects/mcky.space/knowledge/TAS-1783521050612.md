# TAS-1783521050612: B4: Blog code blocks lacked Prism highlighting (marked runtime)

## Bug
astro.config.mjs set syntaxHighlight: 'prism' - but that only applies to Astro's own markdown pipeline. Blog posts are rendered at RUNTIME via marked, which output plain pre/code with no Prism classes. Changelog (2026-07-05) claimed Prism highlighting but it was not applied to blog posts.

## Fix (applied 2026-07-08)
- Installed marked-highlight + prismjs
- src/pages/blog/[slug].astro: marked.use(markedHighlight({...})) + import 'prismjs/themes/prism.css'
- Output now has language-* Prism classes + theme CSS
