# TAS-1783521065777: B7: Add Content-Security-Policy to vercel.json

## Bug
vercel.json set X-Content-Type-Options / X-Frame-Options / Referrer-Policy but NO CSP. Given B1 (stored-XSS sink via marked), CSP is the real containment and was absent. Status/DESIGN docs claimed security headers locked down - incomplete.

## Fix (applied 2026-07-08)
Added to /(.*) headers in vercel.json:
Content-Security-Policy: default-src 'self'; script-src 'self' https://cdn.jsdelivr.net 'unsafe-inline'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; font-src 'self'; connect-src 'self'; frame-ancestors 'none'; base-uri 'self'; form-action 'self'

Note: unsafe-inline is required because Alpine.js store init is inline. script-src allows the Alpine CDN. DOMPurify (B1) remains the primary XSS defense.

DONE - CSP added.
