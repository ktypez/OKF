# TAS-1783521045464: B2: Remove orphaned /api/auth + habits API (client-hash auth spoofable)

## Bug
- /api/auth + /api/habits/* remained deployed after UI auth was stripped.
- src/api/require-auth.ts trusted a client-supplied x-auth-hash header, compared to DB hash. Spoofable bearer token - no signature, no expiry, no server session.
- /api/auth returned the SHA-256 hash on success (leaks the credential).

## Fix (applied 2026-07-08)
Deleted:
- src/pages/api/auth.ts
- src/api/require-auth.ts
- src/lib/supabase.ts
- src/pages/api/habits.ts, habits/[id].ts, habits/toggle.ts, habits/stats.ts, habits/[slug].ts

Site is now fully static. No Supabase dependency remains in main site.
