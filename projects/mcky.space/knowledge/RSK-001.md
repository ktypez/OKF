# RSK-001: Local dev server broken (shiki module corruption)

## Risk

The `npm install` breaks shiki native bindings on android-arm64. Dev server won't start locally.

## Mitigation

- **Shiki fix** (2026-07-04): switched astro config to `syntaxHighlight: 'prism'` — eliminates shiki dependency entirely. Generated missing shiki dist files (`themes.mjs`, `langs-bundle-full-*.mjs`) from individual lang/theme files before the switch.
- **Rolldown fix** (2026-07-04): installed `@rolldown/binding-linux-arm64-gnu@1.1.2` directly — npm's optional dep resolution skips it on Termux. Binding now loads.
- **Unstorage fix** (2026-07-04): generated missing `.mjs` wrapper files for unstorage dist/ and drivers/ (22 files total) — npm install drops ESM files on android-arm64 for dual CJS/ESM packages.
- **Interface enumeration** (2026-07-04): `uv_interface_addresses` EACCES error on Termux — dev server works with `--host 127.0.0.1` instead of `--host` (which tries all interfaces).
- **Dev server is now fully operational** on Termux with the workarounds above.
- Fallback: build/test via Vercel preview deployments if node_modules gets reset.

## Status

Resolved (with workarounds) — all known package-level issues have been addressed. Dev server starts and serves pages. Platform incompatibility with npm optional deps on android-arm64 remains a systemic issue if `node_modules/` is reinstalled from scratch. The manual fixes (shiki files, unstorage wrappers, rolldown binding) must be reapplied after `npm install`.

## Resolution

Closed 2026-07-05. All mitigations in place and dev server fully operational. See TASK-001 (dev environment setup) for task details. Risk is no longer active — systemic android-arm64 npm issue is accepted as a known constraint.
