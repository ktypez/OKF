# DEC-001: In-memory storage for prototype

## Decision

Use in-memory JavaScript objects for all data storage during the experimental prototype phase.

## Rationale

- Fastest path to a working prototype
- No database setup or migration overhead
- Easy to iterate on data shapes

## Consequences

- All data lost on server restart
- Not suitable for production — must migrate to DB
- Cannot scale beyond single-process
