## หลักการเขียน (Writing Principles)

### ถ่อมตัว (Humble)
- ใช้ภาษาธรรมดา ไม่โอเวอร์
- ไม่ต้องยกยอตัวเองหรือโปรเจกต์
- "i built this" แทน "i engineered a robust solution"
- ไม่ใช้คำโฆษณาหรือ hype (cutting-edge, revolutionary, best-in-class)

### เบาๆ แต่พอดี (Light but Informative)
- กระชับ ไม่ยืดเยื้อ แต่**อย่าเบาจนไม่รูเรื่อง**
- อธิบายให้พอเข้าใจ — ไม่ต้องละรายละเอียดที่สำคัญ
- เขียนเหมือนคุยกับเพื่อน แต่เพื่อนต้องเข้าใจสิ่งที่เราพูด
- **บาลานซ์:** ไม่ต้องใช้ภาษาดราม่า แต่ก็ต้องเล่าให้ครบ

### ตัวอย่าง

❌ "We engineered a groundbreaking, state-of-the-art solution that revolutionizes the way you track habits."
✅ "made a habit tracker. it works."

❌ "fixed a bug lol"
✅ "fixed the NaN layout bug — photos without metadata were breaking the grid. fallback to square if dimensions are missing."

❌ "This meticulously crafted design system represents months of careful consideration."
✅ "wrote down the design patterns so i don't forget them."
