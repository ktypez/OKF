# DOC-004: Login modal — default password pre-filled

# ช่อง login ถูก pre-fill รหัสผ่าน `meowmeow` ไว้เลย

**File:** `src/components/LoginModal.tsx`

กล่องรหัสผ่านใน `LoginModal` ถูกตั้งค่าเริ่มต้น (initial state) ให้มีค่า `"meowmeow"` แล้ว — คือ `useState('meowmeow')` แทนที่จะเป็น string ว่าง ผู้ใช้กด "เข้าระบบ" ได้ทันทีโดยไม่ต้องพิมพ์

```tsx
const [password, setPassword] = useState('meowmeow')
```

## พฤติกรรม
- ตอนเปิด modal ช่องจะแสดง `meowmeow` เลย (type=password จึงเป็นจุดดำ)
- ส่ง POST `/api/auth` ด้วย `{ password: "meowmeow" }` ปกติเหมือนเดิม
- เข้าสำเร็จ → เก็บ token ลง `localStorage['ezzylist_admin_token']`

## หมายเหตุ (Manual Notes)
- เจตนา: ให้ทดสอบ/เข้าระบบสะดวก (owner ระบุชัดเจน ไม่ต้องห่วงเรื่องความปลอดภัย)
- ⚠️ ข้อควรระวัง: การ hardcode password เริ่มต้นลงใน client bundle ทำให้ใครก็อ่านได้จาก source/bundle — ห้าม deploy ขึ้น production แบบนี้โดยไม่ตั้งรหัสผ่านใหม่ก่อน
- ถ้าจะ rollback: กลับเป็น `useState('')`
